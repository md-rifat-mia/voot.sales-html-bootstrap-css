# Voot Sales - Outbound Revenue Engine Landing Page

A modern, responsive, and pixel-perfect landing page clone of **Voot Sales**, built using **HTML5**, **Bootstrap**, and custom **CSS3**. This project showcases the structural implementation of a multi-section corporate B2B landing page featuring interactive service matrices, dynamic pricing packages, and strategic calls to action.

## 🛠️ Technologies Used
* **HTML5:** Semantic markup structure.
* **CSS3:** Custom layout styling, color schemes, typography adjustments, and specific component behaviors.
* **Bootstrap:** Component frameworks, grid system alignment, and cross-device responsiveness.
* **Google Fonts & Visual Icons:** Integrated functional iconography and typography mirroring the original landing page design.

## 🌟 Key Features Implemented

* **Hero & Social Proof:** High-conversion headline banner paired with a transparent core metrics ribbon (3-5x meetings, 60% faster launch).
* **The Problem vs. Solution Grid:** Comparative split-column dynamic built for contrasting standard outbound limitations against the Voot process.
* **6-Step Engine Workflow:** A step-by-step ordered matrix layout explaining ideal customer profiling and target validation tracking.
* **Service Framework Cards:** Multi-channel layouts highlighting Cold Email, Tele-Appointing, Signal Prospecting, and RevOps setups.
* **Dual Tier Pricing Matrix:** Clean multi-tier blocks contrasting the "Launch Engine" and the "Revenue Pod" tiers cleanly side-by-side.
* **Full Responsiveness:** Fluid container adaptation making the presentation crisp on mobile displays, tablets, and wide desktop screens.

## 🔧 Installation & Local Setup

If you want to view, modify, or run this project locally on your machine, follow these steps:

1. **Clone the repository:**
   ```bash
   git clone https://github.com
   ```

2. **Navigate into the project directory:**
   ```bash
   cd voot.sales-html-bootstrap-css
   ```

3. **Open the project:**
   * Double-click `index.html` to load it directly in your web browser.
   * Or run it via a code editor extension like Visual Studio Code's **Live Server** for real-time local updates.
